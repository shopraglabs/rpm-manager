## Docs, commits, and pushes

At the end of every session:

1. **Update relevant docs** — mark completed features ✅ in `docs/ROADMAP.md`; add anything built that isn't already in the roadmap; update `CHANGELOG.md` under `[Unreleased]`
2. **Commit** — stage and commit all changed files (including docs) with a clear commit message
3. **Push** — push to `origin/main`

During a session, commit code as work is completed but do not update docs or push until the user signals the session is ending.
