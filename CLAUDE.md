## Docs, commits, and pushes

After completing any meaningful unit of work, you must:

1. **Update relevant docs** — if a feature is completed, mark it ✅ in `docs/ROADMAP.md`; if a new feature is added that isn't in the roadmap, add it; update `CHANGELOG.md` with what was built under `[Unreleased]`
2. **Commit** — stage and commit all changed files (including docs) with a clear commit message
3. **Push** — push to `origin/main`

Do this at the end of every session, or sooner if reaching a natural stopping point mid-session.
